# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do not** open a public issue.
2. Email the maintainer directly or use GitHub's
   [private vulnerability reporting](https://github.com/sgentzen/fallback-cache/security/advisories/new).
3. Include a description of the vulnerability, steps to reproduce, and potential
   impact.

We will acknowledge receipt within 48 hours and aim to provide a fix or
mitigation within 7 days for critical issues.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.2.x   | Yes       |
| < 0.2   | No        |
